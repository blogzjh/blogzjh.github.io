    PreparedStatement实现

    PreparedStatement是Statement的子对象，也是最常用的；原始的Statement无法解决SQL注入问题，所以弃用了。

    作用

    SQL语句被预编译并存储再PreparedStatement对象中，方便使用此对象多次执行

    使用：增

    以增为例：向yeb数据库文件中的t_role表添加一组数据

    • 固定连接操作+PreparedStatement三步操作
    • 开发中一定像IO处理异常一样try-catch而不是简单的throws
    • 开发中可以把固定操作封装成一个util工具
        /**********固定操作******************/
        //读取配置信息:properties——>流
        InputStream is = ConnectionTest1.class.getClassLoader().getResourceAsStream("t_role.properties");
        //还可以写成InputStream is =ClassLoader.getSystemClassLoader().getResourceAsStream("配置文件名");
        Properties pros = new Properties();
        pros.load(is);//配置信息存储到Propertires中

        String user = pros.getProperty("user");
        String password = pros.getProperty("password");
        String url = pros.getProperty("url");
        String driverClass = pros.getProperty("driverClass");


        //加载驱动
        Class.forName(driverClass);

        //获取连接
        Connection conn = DriverManager.getConnection(url, user, password);


    /***************使用PreparedStatement增删改查**************/
        String sql = "insert into t_role(name,nameZh)values(?,?)";//sql操作
        PreparedStatement ps = conn.prepareStatement(sql);//预编译sql语句，返回PreparedStatement实例

        ps.close();
        conn.close();

    Mybatis

    • 封装的JDBC，最简单的持久化框架
    • 灵活，sql语句写在xml里，降低耦合度
    • 提供xml标签，支持编写动态sql语句
    • 提供映射标签，映射对象和数据库的字段(ORM)

    缺点

    • 对SQL语法要求高，编写工作量大
    • SQL语法依赖于数据库，移植性差

    数据源DataSource有什么作用:

    1. 通过DataSource可以获取数据库连接Connection
    2. 通过DataSource创建JdbcTemplate操作数据库

    JdbcTemplate:封装JDBC CRUD

    优点

    1. Spring Boot 默认提供了数据源，默认提供了org.springframework.jdbc.core.JdbcTemplate
    2. JdbcTemplate 中会自己注入数据源，用于简化 JDBC操作
    3. 不用再自己来关闭数据库连接

    JdbcTemplate主要提供以下几类方法：

    • execute方法：可以用于执行任何SQL语句，一般用于执行DDL语句；

    • update方法及batchUpdate方法：update方法用于执行新增、修改、删除等语句；batchUpdate方法用于执行批处理相关语句；

    • query方法及queryForXXX方法：用于执行查询相关语句；

    • call方法：用于执行存储过程、函数相关语句。

    增:update + @GetMapping(“/add/{……

    前端处理注册数据，返回一个url，如：

    localhost:8080/add/{张家豪}/{123456}/{13288888888}/{222222222@qq.com}

    然后GetMapping处理这条url附带的信息

    //增
    @GetMapping("/add/{username}/{password}/{phonenum}/{email}")//格式标准

public String addUser(@PathVariable("username") String name,
                      @PathVariable("password") String pass,
                      @PathVariable("phonenum") String phone,
                      @PathVariable("email") String e){

    //占位符sql语句
    String sql = "insert into userinfo(username,password,phonenum,email) "
            + "values(name,pass,phone,e)";


    //执行sql
    jdbcTemplate.update(sql);


    //返回值
    return "添加成功";

    }
